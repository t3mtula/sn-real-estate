## 🚦 กฎ v2 (ใหม่ — แทนกฎ 6 ข้อของ v1)

1. **Type safety แน่น** — ห้าม `any` ยกเว้นมี comment อธิบายเหตุ
2. **Server state ผ่าน TanStack Query เท่านั้น** — ห้าม fetch ใน component ตรงๆ
3. **Client state ผ่าน Zustand เท่านั้น** — ห้าม Context API ของ React ยกเว้น theme/auth provider
4. **Form ใช้ react-hook-form + zod schema เสมอ** — ห้าม useState สำหรับ form field
5. **CSS = Tailwind utility class** — ห้ามเขียน `.css` file ของ component (ยกเว้น globals.css)
6. **Component ผ่าน shadcn ก่อน** — ก่อนเขียน custom component ต้องเช็คว่า shadcn มีไหม
7. **Server logic ใน Supabase RLS + Edge Functions** — ห้าม sensitive logic ใน client
8. **Format/lint รัน Biome ก่อน commit** — `pnpm biome check --write .`

## 🇹🇭 Thai conventions v2

- **Date display:** พ.ศ. (BE) เสมอใน UI · internal storage = ISO (`YYYY-MM-DD`)
- **Date input:** shadcn `Calendar` component + BE adapter wrapper (`<ThaiDatePicker>`)
- **Money format:** `formatTHB(amount)` util ใน `src/lib/money.ts` — ใช้ `Intl.NumberFormat('th-TH', { style: 'currency', currency: 'THB' })`
- **Language:** ไทยล้วนใน UI · ไม่ใช่ i18n library

## 🔐 Auth — Google Workspace SSO

- Provider: Google OAuth ผ่าน Supabase Auth
- **Restrict: `@sstpconstruction.com` only** (Google Workspace domain ของบริษัท · ทุกคนใน domain login ได้)
- Implementation: ส่ง `hd=sstpconstruction.com` query param ตอน `signInWithOAuth` + Supabase Auth hook (DB trigger) reject email ที่ไม่ใช่ domain นี้
- Logout: Supabase signOut + redirect to `/login`
- Permission: ใช้ Supabase RLS policy + role table ใน Supabase
- Auth guard: ทุก route ยกเว้น `/login` ต้อง authenticated
